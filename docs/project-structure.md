# Project Structure

The generated project keeps the application code small while vendoring the current Xpress++ headers for local development.

```txt
my-api/
  CMakeLists.txt
  main.cpp
  vcpkg.json
  vendor/
    xpresspp/
      include/
```

## Application files

`main.cpp` contains the app entry point and route definitions. `CMakeLists.txt` configures the executable and links native dependencies.

## Vendored framework

The template places Xpress++ headers under `vendor/xpresspp/include` so new applications can build without a global framework install.

## Dependencies

Native packages live in `vcpkg.json`, which currently declares Drogon and Jsoncpp for the generated app.
